## 운용 규칙

1. **토큰이 원본이다.** 모든 색·간격·글꼴 값은 `assets/site.css` 의 `:root` 변수로만 관리한다.
   HTML/CSS에 숫자를 직접 쓰지 않는다.
2. **규칙을 바꾸면 세 곳을 같이 고친다.** `assets/site.css` 토큰 → 이 디렉토리의 해당 문서 → 관리자 KMS 디자인 룰북.
3. **문서와 코드가 어긋나면 코드가 아니라 규칙을 의심한다.** 어긋난 사실을 먼저 보고하고, 어느 쪽을 맞출지 정한 뒤 고친다.
4. **예외를 만들 때는 이유를 문서에 남긴다.** 이유 없는 예외는 다음 사람에게 그냥 '기존 패턴'으로 복제된다.

## KMS 디자인 룰북과의 관계

관리자 콘솔의 **KMS > 디자인 룰북**(`assets/admin.js` 의 기본 문서, localStorage 저장)은
**고객사 열람·기록용 사본**입니다. 규칙의 원본은 이 디렉토리이며, 룰북은 여기 내용을 따라 갱신합니다.

> 2026-07-21 기준 KMS 룰북 본문은 일부 항목이 실제 코드와 어긋나 있습니다
> (타이포 스케일, `--sec-pad` 고정값 80px, 내비 높이 76px, 섹션 내부 간격 규칙 부재).
> 갱신 여부는 고객사 확인 후 반영합니다.
