# 세로 리듬 (간격 규칙)

간격은 이 사이트에서 가장 자주 어긋나는 항목입니다. **허용 값을 좁게 정의하고 그 밖의 값은 쓰지 않습니다.**

---

## 1. 허용 간격 (이 5개만 쓴다)

| 용도 | 토큰 | 값 |
|---|---|---|
| 밀접 — 라벨↔값, 아이콘↔글자 | `--gap-tight` | `8px` |
| 관련 — 제목↔문단, 문단↔문단 | `--gap-related` | `16px` |
| 블록 — 카드 안 요소, 목록 항목 사이 | `--gap-block` | `24px` |
| 블록군 — 섹션 헤드↔본문, 서로 다른 덩어리 | `--gap-group` | `clamp(28px, 3.4vw, 44px)` |
| 소제목 — 섹션 안 `h3` 앞 | `--gap-sub` | `clamp(40px, 5vw, 64px)` |

4px 그리드(`--s1`~`--s32`)는 컴포넌트 내부 여백용으로 그대로 두고,
**블록 사이 세로 간격에는 위 5개 토큰만** 씁니다.
6 / 14 / 18 / 20 / 22 / 26 / 28 / 30 / 32 / 34 / 36 / 40 / 48 / 52 / 56 / 72px 같은 중간값은 쓰지 않습니다.

> `--gap-group` 은 4px 그리드 위의 값이지만 `--s10(40)` · `--s12(48)` 어느 쪽도 아닌 44px입니다.
> 기존 코드에서 이미 가장 많이 쓰이던 값(28건 중 10건)이라 변경 폭이 가장 작고, 헤드와 본문 사이
> 무게감이 가장 자연스러워 전용 리듬 토큰으로 승격했습니다.

## 2. 섹션 상하 여백

```css
--sec-pad: clamp(58px, 7vw, 116px);   /* 390px→58 · 1280px→90 · 1660px 이상→116 */
```

섹션 사이가 좁으면 페이지 전체가 답답하게 읽힙니다. 위 값은 실측으로 정한 하한이며,
**줄이지 않습니다.**

- `.section` — `padding-block: var(--sec-pad)`
- `.section-sm` — `var(--sec-pad) * 0.62` (CTA·신뢰 스트립처럼 밀도 높은 섹션)
- **고정 px 금지.** 고정값이면 모바일에서도 데스크톱 여백이 그대로 적용돼 페이지가 과하게 길어집니다.

### 같은 배경이 이어질 때

앞 섹션의 아래 패딩과 뒤 섹션의 위 패딩이 겹쳐 **경계도 없는 2배 높이의 빈 띠**가 생깁니다.
→ 뒤 섹션의 위 패딩을 제거해 1배로 맞춥니다.

```css
:is(.hero, .partners) + :is(.section, .section-sm):not([class*="band-"]),
:is(.section, .section-sm):not([class*="band-"]) + :is(.section, .section-sm):not([class*="band-"]) {
  padding-top: 0;
}
```

색 밴드(`.band-oat` / `.band-tint` / `.band-deep` / `.band-sage`)가 바뀌는 경계는
**배경 전환 자체가 구분선**이므로 양쪽 패딩을 그대로 둡니다.

## 3. 섹션 내부 리듬

```
.section
 └ .wrap
    ├ .section-head          (eyebrow → h2 16px → lede 16px)
    ├ [본문 블록]             ← 헤드에서 --gap-group (44px)
    ├ h3 소제목               ← 앞 요소에서 --gap-sub (64px)
    ├ [본문 블록]             ← h3에서 --gap-block (24px)
    └ 각주 / 부연              ← 앞 블록에서 --gap-related (16px)
```

이 리듬은 **리듬 컨테이너**가 자동으로 적용합니다. HTML에서 다시 지정하지 않습니다.

| 컨테이너 | 직계 블록 사이 | 비고 |
|---|---|---|
| `.wrap` | `--gap-group` | 섹션 본문. `.wrap.split` 은 2열 레이아웃이라 제외 |
| `.stack` | `--gap-group` | 2단 안의 글 기둥처럼 `.wrap` 이 아닌 덩어리에 붙인다 |
| `.card-pad` | `--gap-related` (제목 뒤는 `--gap-tight`) | 카드 내부 |
| `.page-hero .wrap` | `--gap-related` | 제목 묶음이라 본문 리듬을 쓰지 않는다 |

```css
.wrap:not(.split) > * + *,  .stack > * + *  { margin-top: var(--gap-group); }
.wrap:not(.split) > h3,     .stack > h3     { margin-top: var(--gap-sub); }
.wrap:not(.split) > h3 + *, .stack > h3 + * { margin-top: var(--gap-block); }
.wrap:not(.split) > p + p,  .stack > p + p  { margin-top: var(--gap-related); }
.section-head + *                           { margin-top: var(--gap-group); }
```

**이어지는 문단끼리는 `--gap-related`(16px)** 입니다. 여러 문단이 이어지는 글기둥
(인사말·파트너 소개)에서 문단마다 44px씩 벌어지면 한 편의 글이 아니라 낱개 조각으로 읽힙니다.

**키커 → 제목 → 리드문 묶음은 `.section-head` 로 감쌉니다.**
리듬 컨테이너 안에서 감싸지 않으면 낱개 블록으로 취급돼 44px씩 벌어집니다.
컨테이너 밖(예: `.split` 안의 열)에서는 아래 대체 규칙이 같은 리듬을 만듭니다.

```css
.eyebrow + :is(h1, h2, h3) { margin-top: var(--gap-related); }
:is(h1, h2, h3) + .lede    { margin-top: var(--gap-related); }
```

- `.section-head` 내부(eyebrow→h2→lede)도 CSS가 이미 16px로 잡고 있습니다.
- 그리드(`.grid`)·2단(`.split`)·에디토리얼(`.spec`)의 내부 간격은 컴포넌트 CSS가 담당합니다.
- 리듬에서 벗어나야 하는 자리가 생기면 **그 컴포넌트의 CSS에 규칙을 추가**합니다.
  HTML에 인라인 margin을 되돌리지 않습니다.

## 4. 좌우 여백

```css
--gutter: clamp(18px, 4.2vw, 32px);   /* .wrap 의 padding-inline */
```

모바일에서 32px 고정은 390px 화면의 본문 폭을 16%나 깎습니다.

## 5. 인라인 스타일 금지

HTML에 `style="margin-top:44px"` 같은 값을 직접 쓰지 않습니다.
인라인 스타일은 **미디어쿼리보다 우선순위가 높아 반응형 조정을 원천 차단**합니다.
(실제로 가격표의 `<col style="width:20%">` 때문에 모바일 열 너비 조정이 먹지 않는 사고가 있었습니다.)

간격이 필요하면 컴포넌트 CSS 클래스로 표현합니다.

## 정리 이력 (2026-07-21, Playwright 1280px 실측)

공개 페이지 9개를 실측해 인라인 margin 124건을 걷어내고 위 규칙으로 수렴시켰습니다.

| 위치 | 정리 전 | 정리 후 |
|---|---|---|
| 섹션 헤드 → 본문 (29건) | 8가지 (16·32·36·40·44·48·52·56) | **1가지 — 44px** |
| `h3` 소제목 앞 (4건) | 2가지 (64·72) | **1가지 — 64px** |
| 블록 사이 (69건) | **18가지** (6~72px) | **3가지 — 16 · 24 · 44px** |

같은 역할의 간격이 페이지마다 다른 값으로 들어가 있던 것이, 페이지를 옮겨 다닐 때
리듬이 어긋나 보이던 원인이었습니다.

### 회귀 검사

간격을 건드린 뒤에는 위 표의 "값 종류"가 늘어나지 않았는지 확인합니다.
허용 값(8 · 16 · 24 · 44 · 64) 밖의 숫자가 나오면 인라인 margin이 다시 들어간 것입니다.
