# 세로 리듬 (간격 규칙)

간격은 이 사이트에서 가장 자주 어긋나는 항목입니다. **허용 값을 좁게 정의하고 그 밖의 값은 쓰지 않습니다.**

---

## 1. 허용 간격 (이 5개만 쓴다)

| 용도 | 값 | 토큰 |
|---|---|---|
| 밀접 — 라벨↔값, 아이콘↔글자 | 8px | `--s2` |
| 관련 — 제목↔문단, 문단↔문단 | 16px | `--s4` |
| 블록 — 카드 안 요소, 목록 항목 사이 | 24px | `--s6` |
| 블록군 — 섹션 헤드↔본문, 서로 다른 덩어리 | 40px | `--s10` |
| 소제목 — 섹션 안 `h3` 앞 | 64px | `--s16` |

4px 그리드(`--s1`~`--s32`)는 그대로 두되, **세로 간격에 실제로 쓰는 값은 위 5개**입니다.
14 / 18 / 20 / 26 / 28 / 30 / 34 / 36 / 44 / 48 / 52 / 56 / 72px 같은 중간값은 쓰지 않습니다.

## 2. 섹션 상하 여백

```css
--sec-pad: clamp(46px, 5.6vw, 80px);   /* 390px→46 · 1280px→72 · 1440px 이상→80 */
```

- `.section` — `padding-block: var(--sec-pad)`
- `.section-sm` — `var(--sec-pad) * 0.62` (CTA·신뢰 스트립처럼 밀도 높은 섹션)
- **고정 px 금지.** 고정값이면 모바일에서도 데스크톱 여백이 그대로 적용돼 페이지가 과하게 길어집니다.

### 같은 배경이 이어질 때

앞 섹션의 아래 패딩과 뒤 섹션의 위 패딩이 겹쳐 **경계도 없는 2배 높이의 빈 띠**가 생깁니다.
→ 뒤 섹션의 위 패딩을 제거해 1배로 맞춥니다.

```css
:is(.hero, .partners) + :is(.section, .section-sm):not([class*="band-"]),
:is(.section, .section-sm):not([class*="band-"]) + :is(.section, .section-sm):not([class*="band-"]) {
  padding-top: 0;
}
```

색 밴드(`.band-oat` / `.band-tint` / `.band-deep` / `.band-sage`)가 바뀌는 경계는
**배경 전환 자체가 구분선**이므로 양쪽 패딩을 그대로 둡니다.

## 3. 섹션 내부 리듬

```
.section
 └ .wrap
    ├ .section-head          (eyebrow → h2 16px → lede 16px)
    ├ [본문 블록]             ← 헤드에서 40px
    ├ h3 소제목               ← 앞 요소에서 64px
    ├ [본문 블록]             ← h3에서 24px
    └ 각주 / 부연              ← 앞 블록에서 16px
```

- `.section-head` 내부(eyebrow→h2→lede)는 CSS가 이미 16px로 잡고 있습니다. HTML에서 다시 지정하지 않습니다.
- 그리드(`.grid`)·2단(`.split`)·에디토리얼(`.spec`)의 내부 간격은 컴포넌트 CSS가 담당합니다.

## 4. 좌우 여백

```css
--gutter: clamp(18px, 4.2vw, 32px);   /* .wrap 의 padding-inline */
```

모바일에서 32px 고정은 390px 화면의 본문 폭을 16%나 깎습니다.

## 5. 인라인 스타일 금지

HTML에 `style="margin-top:44px"` 같은 값을 직접 쓰지 않습니다.
인라인 스타일은 **미디어쿼리보다 우선순위가 높아 반응형 조정을 원천 차단**합니다.
(실제로 가격표의 `<col style="width:20%">` 때문에 모바일 열 너비 조정이 먹지 않는 사고가 있었습니다.)

간격이 필요하면 컴포넌트 CSS 클래스로 표현합니다.

## 현황 (2026-07-21 실측, Playwright 1280px)

규칙 도입 이전의 공개 페이지 9개 실측치입니다. **정리 대상**입니다.

| 위치 | 값 종류 | 실제로 쓰인 값 |
|---|---|---|
| 섹션 헤드 → 본문 (28건) | 8가지 | 16 · 32 · 36 · **40** · **44** · 48 · 52 · 56 |
| `h3` 소제목 앞 (4건) | 2가지 | 64 · 72 |
| 블록 사이 (73건) | **18가지** | 6 · 8 · 14 · 16 · 18 · 20 · 22 · 24 · 26 · 28 · 30 · 32 · 34 · 36 · 40 · 44 · 48 · 72 |

같은 역할의 간격이 페이지마다 다른 값으로 들어가 있어, 페이지 사이 리듬이 어긋나 보이는 원인입니다.
