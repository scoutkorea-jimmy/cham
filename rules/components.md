# 컴포넌트 표준

새 컴포넌트를 만들기 전에 **같은 역할의 기존 컴포넌트를 먼저 찾습니다.**
비슷한데 이름만 다른 컴포넌트를 늘리지 않습니다.

---

## 1. 섹션 아키타입

내용의 성격에 따라 형태를 달리해 단조로움을 없앱니다. 아무 섹션에나 카드 그리드를 쓰지 않습니다.

| 클래스 | 쓰는 곳 |
|---|---|
| `.pillars` | 대표 진입 3~4개 — 사진 위 텍스트, 큰 면적 |
| `.rail` | **순서가 핵심인 내용** (식초 제조 4단계, 발효 과정 5단계) — 점선으로 연결 |
| `.spec` | 긴 설명형 — 사진과 글을 좌우 교차 배치 |
| `.quote` | 대표 말씀·브랜드 문장 — 본문과 다른 무게 |
| `.chips` | 항목이 많고 각각은 짧은 목록 (강의처, 라인업) |
| `.facts` | 라벨/값 2열 — 정보 밀도 높은 안내표 |
| `.trust` | 숫자 근거 나열 (인증·기수·연도) |
| `.quick` | 진입 타일 — '무엇을 찾으세요?' |
| `.jump` | 긴 페이지의 목차 바 (sticky, 현재 위치 표시) |

## 2. 배경 밴드

| 클래스 | 배경 | 쓰임 |
|---|---|---|
| (없음) | 흰색 | 기본 지면 |
| `.band-tint` | `--surface-2` | 가벼운 구분 |
| `.band-oat` | `--sub-soft` | 따뜻한 구분 — 이야기·소개 |
| `.band-deep` | `--main-deeper` | 강한 전환 — 교육·씨장 등 무게 있는 내용 |
| `.band-sage` | `--olive` | CTA — 문의·신청 |

- **같은 밴드를 연달아 쓰지 않는다.** 구분이 사라지고 여백만 두 배가 된다
  (→ [spacing-rhythm.md](spacing-rhythm.md) 2절).
- 밴드 위 텍스트 색은 CSS가 자동 반전한다. HTML에서 색을 직접 지정하지 않는다.

## 3. 버튼

pill형(`--r-pill`). 한 화면에 **핵심 CTA는 하나**.

| 클래스 | 쓰임 |
|---|---|
| `.btn-point` | 핵심 CTA (옐로우 면 + `--point-ink` 글자) |
| `.btn-primary` | 일반 주요 동작 |
| `.btn-ghost` | 보조 동작 |
| `.btn-on-dark` | 짙은 배경 위 |
| `.btn-text` | 인라인 링크형 ('전체 보기 →') |

`.btn-lg` 는 섹션 CTA에만. 카드 안에서는 기본 크기를 쓴다.

## 4. 카드

`--surface` + `1px --line-soft` + `--r-lg` + `--sh-sm`, 호버 시 3px 상승 + `--sh-md`.
내부 여백은 `.card-pad` (`clamp(20px, 3vw, 32px)`) 를 쓰고 직접 padding을 쓰지 않는다.

## 5. 표

| 클래스 | 쓰임 |
|---|---|
| `.table` + `.table-wrap` | 일정표 등 일반 표. 넓으면 wrap이 가로 스크롤 |
| `.ptable` | 가격표 — 판매가 고지용 |

### 가격표 열 너비 규칙

`table-layout: auto` 는 **남는 폭을 '내용이 넓은 열'에 비례 배분**해 품목/용량 사이가 크게 벌어집니다.
→ `table-layout: fixed` + `<colgroup>` 의 열 클래스로 고정하고, 남는 폭은 **우측 정렬된 판매가 열이 흡수**합니다.

```html
<colgroup><col class="c-cat"><col class="c-item"><col class="c-vol"><col class="c-price"></colgroup>
```

| 열 클래스 | 데스크톱 | 모바일(≤640) |
|---|---|---|
| `.c-cat` (구분) | 14% | 20% |
| `.c-item` (품목) | 32% (구분 열 없으면 46%) | 32% (없으면 50%) |
| `.c-vol` (용량) | 20% | 24% |
| `.c-price` (판매가, 우측 정렬) | 34% | 24% |

- **`<col>` 에 인라인 `style="width:…"` 를 쓰지 않는다.** 인라인 스타일이 미디어쿼리를 이겨서 모바일 조정이 먹지 않는다.
- 확정 판매가가 없는 품목은 숫자 대신 `가격 문의`(`.price-ask`)로 표기한다. **미확정 가격을 노출하지 않는다.**

## 6. 폼 · 동의

- 지도사 신청 / 제품 주문 / 문의 — **동의 체크 시에만 제출 버튼 활성화**
- 동의문 전문은 관리자 > 동의문 관리에서 관리 (하드코딩 금지)
- 폼 그리드 `.form-grid` 2열, 640px 이하 1열

## 7. 이미지 슬롯

- `.ph` — 사진이 아직 없는 자리의 색면 플레이스홀더. `data-label` 로 어떤 사진이 들어갈 자리인지 표기
- 실제 사진이 들어가면 `.has-img` 로 placeholder 요소를 감춘다
- 비율은 `.ratio-1` / `.ratio-43` / `.ratio-32` / `.ratio-169` / `.ratio-photo` 중에서 고른다
- 제품 병 사진처럼 **세로로 긴 피사체는 `.prod-photo`**(contain)로 잘라내지 않는다

## 8. 이름 규칙

- 컴포넌트 내부 요소는 짧은 접두사: `.t-num` `.qi-ico` `.p-go` `.r-meta` `.c-item`
- `new` / `old` / `v2` / `final` / `copy` / `temp` 같은 시간 접미사를 쓰지 않는다
