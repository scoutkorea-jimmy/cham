# 반응형 규칙

## 1. 기본 원칙

1. **먼저 유동값(`clamp`), 그다음 브레이크포인트.**
   간격·글꼴은 `clamp()` 로 연속 축소하고, 미디어쿼리는 **레이아웃이 바뀌는 지점**(열 수 변경, 요소 표시/숨김)에만 쓴다.
   고정 px + 미디어쿼리 조합은 브레이크포인트 사이에서 어색한 값을 만든다.
2. **가로 스크롤은 언제나 결함이다.** `document.scrollWidth > 뷰포트` 면 고쳐야 한다.
   예외는 `.table-wrap` / `.table-scroll` 안에서 의도적으로 스크롤되는 넓은 표뿐이다.
3. **중요한 정보를 스크롤 뒤로 숨기지 않는다.**
   가격표를 가로 스크롤로 두면 정작 판매가가 화면 밖으로 밀린다 → 열 비율을 다시 잡아 전부 보이게 한다.
4. **모바일에서 정보를 삭제하지 않는다.** 배치를 바꿔 담는다.

## 2. 브레이크포인트

| 폭 | 바뀌는 것 |
|---|---|
| `1080px` | 내비 항목 패딩·글자 축소 — 7개 메뉴를 한 줄로 유지 |
| `960px` | `.g-5` `.g-4` `.g-3` → 2열 |
| `900px` | **내비 → 햄버거**, `--nav-h` 132→80px, 히어로 1열, `.pillars` 1열, `.rail` 2열 |
| `880px` | 상품 상세 1열 + 하단 고정 구매바(`.buybar`) |
| `860px` | `.split` 1열, `.spec` 1열, `.quick` 2열, `.trust` 2열, 목차 바(`.jump`) 상단 위치 |
| `820px` | 푸터 1열 + 링크 터치 영역 확대 |
| `640px` | 모든 그리드 1열, 폼 1열, 가격표 열 비율 재조정 |
| `600px` | 브랜드 로고·법인명 축소 (한 줄 유지) |
| `520px` | `.rail` 1열 |
| `430px` | 브랜드 추가 축소, 상단 CTA 숨김(햄버거 메뉴의 신청 버튼 사용) |

> 브레이크포인트가 10개로 많다. 새 항목을 추가할 때는 **새 지점을 만들기보다 기존 지점에 붙인다.**
> 특히 860/880/900 은 역할이 인접하므로 통합 여지가 있다.

### 뷰포트가 아니라 그릇이 결정하는 것 — 컨테이너 쿼리

같은 컴포넌트가 본문 전폭에도, 2열 카드 안에도 들어간다면 접히는 기준은 **뷰포트가 아니라 그 컴포넌트가 놓인 폭**이다.
뷰포트로 판단하면 넓은 화면의 좁은 카드 안에서 데스크톱 레이아웃이 그대로 남아 내용 최소폭이 카드를 밀어내고 가로 스크롤이 생긴다.

- `.facts` — `container-type: inline-size` + `@container (max-width: 420px)` 로 1열 전환.
  420px = 라벨 열 180px + 간격 24px 을 빼고도 값 열에 200px 이상 남는 최소 폭.
  (`about.html` #people 의 2열 카드 안에서 180px 라벨 열이 카드를 밀어내 768px에서 가로 스크롤 52px이 생기던 문제)

새 컴포넌트도 **여러 폭의 그릇에 들어갈 수 있다면** 미디어쿼리 대신 컨테이너 쿼리를 쓴다.

## 3. 터치 영역

- 링크·버튼의 **터치 높이 최소 32px**, 가능하면 40px 이상.
- 데스크톱에서 17px 높이인 텍스트 링크(푸터 등)는 모바일에서 `padding-block` 으로 넓힌다.
- 예외: 지도(Leaflet) 기본 컨트롤 등 외부 라이브러리 UI.

## 4. 검증 방법

`.claude/skills/verify` 의 절차대로 실제 브라우저로 확인한다. 최소 확인 항목:

- **390px**(모바일 기준폭) · **1280px**(노트북) 두 폭
- 가로 스크롤 0건
- 섹션 경계의 빈 띠가 170px를 넘지 않을 것
- 표의 마지막 열(가격 등 핵심 정보)이 화면 안에 있을 것
- 터치 영역 32px 미만 요소 없을 것
