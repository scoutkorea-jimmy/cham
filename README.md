# 한국찬전통발효식품협동조합 — 홈페이지

전통 발효식품의 보급·교육·판매를 위한 협동조합 공식 홈페이지입니다.
순수 **HTML · CSS · 바닐라 JS** 로 구성되어 별도 빌드 과정 없이 그대로 정적 서버에 배포할 수 있습니다.

> 현재 버전은 **기능 검증용 목업(데모)** 입니다. 고객사 승인 후 데이터 계층을 서버 API로 교체하여 운영 서버에 배포합니다.

---

## 실행 / 배포

빌드 도구가 필요 없습니다. 정적 파일 서버에 올리면 됩니다.

```bash
# 로컬 미리보기 (둘 중 아무거나)
npx serve .
python3 -m http.server 8000
```

`index.html` 이 진입점입니다. GitHub Pages / Netlify / Vercel / Nginx 등 어떤 정적 호스팅에도 그대로 배포 가능합니다.

> 외부 CDN 의존성: Pretendard·페이북 글꼴, Lucide 아이콘, Tiptap v2(esm.sh — 게시판 에디터). 폐쇄망 배포 시 로컬 호스팅으로 교체하세요.

## 파일 구조

```
/
├─ index.html          메인 (Hero · 파트너 무한슬라이드 · 핵심가치 · 체험지도사 · 제품 · 소식)
├─ about.html          협동조합 소개 (인사말 · 설립목적 · 비전 · 파트너 정선만장대 · 오시는 길)
├─ ferments.html       전통발효식품 (발효 원리 · 씨장 · 종류 · 5단계 과정)
├─ instructor.html     전통발효식품체험지도사 과정 (커리큘럼 · 혜택 · 일정 · 신청)
├─ products.html       제품 목록 (장류 · 발효식품 · 씨장 분양 · 선물세트 · 비회원 주문조회)
├─ product.html        상품 상세페이지 (?id= 파라미터 — 갤러리 · 옵션 · 구매 · 상품정보고시)
├─ news.html           소식마당 (공지 · 교육 게시판 — Tiptap 에디터 · 현장 갤러리)
├─ contact.html        문의하기
├─ admin.html          관리자 콘솔
├─ robots.txt / sitemap.xml / llms.txt   SEO · AI 검색 대응 (도메인 확정 시 URL 교체)
└─ assets/
   ├─ site.css         디자인 시스템 + 전 컴포넌트 스타일
   ├─ site.js          공통 셸(내비/푸터) · 모달 · 스토어 · 비회원 주문 · 동의 게이팅
   ├─ shop.js          제품 목록 · 상품 상세페이지
   ├─ board.js         게시판(Tiptap) · 첨부파일 · 갤러리
   ├─ admin.js         관리자 콘솔 로직
   ├─ logo.png         로고
   └─ partner-manjangdae-poster.jpg   파트너(정선만장대) 소개 이미지
```

공통 **네비게이션·푸터** 는 `site.js` 가 각 페이지의 `<header id="site-nav">` / `<footer id="site-footer">` 자리에 주입합니다. 메뉴는 `site.js` 상단의 `NAV` 배열에서 수정합니다.

## 주요 기능

### 쇼핑 (비회원 기반 — 회원가입·로그인 없음)
- **상품 상세페이지**: 대표/추가 이미지 갤러리(모바일 스와이프), 옵션·수량 선택 → 금액 자동 계산, 품절 시 구매 버튼 비활성화, 모바일 하단 고정 구매바
- **법적 고지**: 상품정보고시(식품 필수 항목: 원재료·소비기한·보관방법) · 배송안내 · 교환/반품/환불 안내 — 아코디언, 관리자에서 상품별 입력 또는 기본 템플릿 불러오기
- **비회원 주문**: 무통장입금 안내 + 주문번호 발급, **주문번호 + 연락처(또는 이메일)** 로 주문 조회
- **관련 상품**: 관리자가 상품별로 직접 지정
- **주문 관리(관리자)**: 상태 탭(주문접수→결제완료→배송준비중→배송중→배송완료, 취소·반품·교환) + 일괄 처리 — 발송처리(운송장), 배송준비, 입금확인, 되돌리기, 강제 배송완료, 판매취소, 반품/교환 처리

### 게시판 (소식마당)
- 글쓰기 버튼 → **관리자 로그인 시에만 활성화**
- **Tiptap v2 에디터** (무료 확장 전체: 제목/굵게/밑줄/형광펜/정렬/목록/체크리스트/인용/코드/표/이미지/유튜브/글자색 등)
- 첨부파일 **최대 10개 × 개당 10MB, 형식 제한 없음** — 첨부 이미지는 게시글 하단 슬라이드로 미리보기
- 현장 갤러리: **페이지당 최대 10장** 페이지네이션

### 개인정보 동의
- 지도사 신청 / 제품 주문(+ 제3자 제공) / 문의 양식 모두 **동의 체크 시에만 제출 버튼 활성화**
- 동의문 전문은 **관리자 > 동의문 관리** 에서 수정 (표준안 기본 탑재)
- 회원 계정·아이디·이메일을 수집/관리하지 않습니다 (이메일은 주문 조회용 선택 입력만)

### 관리자 콘솔 (`admin.html`)
- 대시보드(오늘 방문자·페이지뷰·오늘 주문·입금 대기, 최근 7일 방문 추이) · 상품 관리(등록/수정/옵션/이미지 즉시 미리보기/**Tiptap 상세설명**/관련상품 드롭다운 다중선택/고시) · 주문 관리 · 지도사 신청 · 문의 · 게시글 · 파트너(가로형 로고) · 팝업(이미지 업로드) · **동의문(서브탭)** · **KMS(서브탭)** · **데이터 백업**
- 접속: 푸터 우측 하단 '관리자' 링크. **계정 정보는 내부 채널로 별도 전달** (문서·화면·소스에 평문 표기하지 않음)

### KMS (지식관리)
- **표준 KMS**(개발 규칙·원칙) / **디자인 룰북**(디자인 표준) 두 문서를 서브탭으로 관리.
- 기본은 서식이 입혀진 **보기(렌더) 모드** — 제목·섹션·불릿을 정리하고, 디자인 룰북의 색상 토큰(`#RRGGBB`)은 색상칩으로 표시. '편집'을 누르면 원문 편집(textarea)으로 전환됩니다.

### 데이터 백업 · 복원
- 모든 데이터는 이 브라우저(localStorage + IndexedDB)에만 저장됩니다. **관리자 > 데이터 백업** 에서 전체(데이터 + 이미지)를 JSON 한 파일로 **내보내기/가져오기** 할 수 있어, 기기 이전·백업·복원이 가능합니다.

### 인증·보안 (데모 모델)
- **로그인 상태를 저장하지 않음** — 새로고침/이동 시 재인증, 공개 페이지의 관리 버튼은 클릭할 때마다 인증을 요구합니다.
- 자격증명은 평문이 아닌 **SHA-256 해시**로 비교하며, **5회 실패 시 5분 잠금**(무작위 대입 방지).
- 클라이언트 검증이므로 한계가 있습니다 — 운영 시 반드시 **서버 세션/토큰 인증 + HTTPS + 서버측 레이트리밋**으로 교체하세요. (가이드: 관리자 > KMS > 표준 KMS)

### 약관 · 개인정보처리방침
- `terms.html`(이용약관), `privacy.html`(개인정보처리방침) — 표준 양식 기반 초안이며, 푸터에서 링크됩니다. **시행 전 법무 검토 후 확정**하세요.

### 오시는 길
- `about.html` 하단 지도는 **OpenStreetMap(Leaflet)** 으로 연동했습니다. 정확한 핀은 `about.html` 의 `LAT`/`LNG` 두 값만 교체하면 됩니다.

### 성능 · 안정성
- 외부 스크립트 **버전 고정 + `defer` + `preconnect`**(렌더 블로킹 제거), 로고 경량화(208KB→120KB), 이미지 **ObjectURL 누수 정리**, **localStorage 쿼터 초과 시 안내**, IndexedDB 미지원(사적 모드)·에디터 로드 실패 시 **graceful degrade**.

## 디자인 시스템

모든 색·간격·폰트는 `assets/site.css` 상단 `:root` 의 CSS 변수로 관리되며, 상세 기준은 **관리자 > KMS > 디자인 룰북** 에 기록·관리됩니다.

| 토큰 | 값 | 용도 |
|---|---|---|
| `--main` | `#6E8252` | 메인(햇살 들녘 올리브그린) |
| `--sub` | `#E2D9BE` | 서브(오트밀) |
| `--point` | `#B0473A` | 포인트(대추) |
| `--bg` | `#FCFAF3` | 배경 |
| `--font-body` / `--font-head` | PayboocFont | 본문/제목 글꼴 |

배경 단색면의 **전통 칠보문(七寶紋) 패턴** 은 `--pat-dark` / `--pat-light` 변수로 제어하며, `<html class="pat-off">` 또는 `pat-strong` 으로 강도를 조절할 수 있습니다.

## 데이터 계층 (현재: 프런트엔드 데모 → 운영 전환 지점)

| 데이터 | 저장소 | 키 / 스토어 |
|---|---|---|
| 주문 (제품·씨장 분양) | localStorage | `kach_orders` |
| 체험지도사 신청 | localStorage | `kach_applications` |
| 문의 | localStorage | `kach_inquiries` |
| 게시글 | localStorage | `kach_posts_v1` |
| 상품 | localStorage | `kach_products_v2` |
| 파트너 / 팝업 / 동의문 / KMS / 방문통계 | localStorage | `kach_partners_v1` `kach_popups_v1` `kach_consents_v1` `kach_kms_v1` `kach_visits_v1` |
| 게시글 첨부 / 갤러리 사진 / 상품 이미지 | IndexedDB `kach_db` | `files` / `gallery` / `pimg` |

**백엔드 연동 포인트**: `site.js` 의 `submitModal()`, `admin.js` 의 각 저장 호출을 `fetch()` API 로 교체하면 됩니다. 관리자 인증은 반드시 서버 세션/토큰 방식으로 교체하세요.

## SEO · AI 검색 대응

- 전 페이지 `title` · `description` · Open Graph 메타 + JSON-LD(Organization · BreadcrumbList · Product)
- `robots.txt`(admin 제외) · `sitemap.xml` · `llms.txt` — **도메인 확정 시 내부 URL을 실제 도메인으로 교체**
- `canonical` / `og:url` 은 `site.js` 가 런타임에 자동 주입

- [ ] `robots.txt` / `sitemap.xml` / `llms.txt` 도메인 교체
- [ ] 방문 통계 → 서버 로그 또는 애널리틱스 연동
